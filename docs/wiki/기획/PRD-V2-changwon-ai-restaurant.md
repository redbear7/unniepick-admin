# PRD: 언니픽 V2 · 창원 AI 맛집 추천 시스템

**Author**: bangju
**Date**: 2026-04-17
**Status**: Draft
**Target**: 언니픽 앱 V2 (V1 런칭 후 추가)
**Tagline**: *"창원 최초 AI 맛집 추천 시스템"*

---

## 1. Executive Summary

창원 지역 맛집 50만+ 데이터 포인트를 축적해 **상황/감정/동행 맥락**을 이해하고 대화로 맛집을 추천하는 AI 챗봇 서비스. 네이버·카카오 지도가 제공하지 못하는 *"오늘 데이트 분위기 좋은 이자카야"* 수준의 맥락 기반 추천을 창원 지역 최초로 상용화한다.

**한 줄 가치**: 지도 앱이 답을 찾게 하는 동안, 언니픽은 대화로 "딱 맞는 곳"을 제안한다.

---

## 2. Background & Context

### 현재 시장의 한계
| 서비스 | 한계 |
|--------|------|
| 네이버 플레이스 | 키워드 검색 기반, 맥락 이해 불가 ("데이트 분위기 좋은" → 안 나옴) |
| 카카오맵 | 거리/평점 정렬 중심, 상황별 추천 부재 |
| 다이닝코드/블루리본 | 서울/부산 중심, 창원 커버리지 취약 |
| 미쉐린 | 창원 미포함 |

### 언니픽만의 강점
- **B2B 매장 네트워크 보유** (V1): 매장 사장님 연결 고리 → 쿠폰/예약/정보 업데이트 협력 가능
- **1인 개발자의 민첩성**: 대기업 플랫폼 대비 빠른 창원 특화 기능 반영
- **이미 구축된 빅데이터 파이프라인**:
  - 크롤러 (일 50+ 업체, 24개 동 대응 가능)
  - Supabase + Metabase + Directus
  - 리뷰/블로그 감성 데이터
  - 텔레그램 신규 업체 알림

### V1 (현재) vs V2 (목표)
| | V1 (2026-04 런칭) | V2 (Target: 2026 Q3) |
|--|------------------|---------------------|
| 서비스 | BGM 스트리밍 + TTS (B2B) | B2B + 창원맛집 탭 (B2C) |
| 사용자 | 매장 사장님 | 사장님 + 일반 소비자 |
| 수익모델 | 구독료 | 구독 + 맛집 광고/쿠폰 수수료 |

## 3. Objectives & Success Metrics

### Goals
1. **창원 1위 맛집 추천 앱** 포지셔닝 (앱스토어 "창원 맛집" 키워드 1페이지 노출)
2. **대화 UX로 차별화**: 버튼/필터가 아닌 자연어 대화로 90%+ 추천 성공 경험 달성
3. **B2B→B2C 확장**: V1 사장님 매장을 AI 추천 대상에 연동 (언니픽 매장 우선 노출)

### Non-Goals
1. ❌ 전국 서비스 — 창원/마산/진해만 집중 (로컬 깊이 우선)
2. ❌ 배달/예약 직접 처리 — 네이버/카카오 링크로 연결
3. ❌ 일반 음식점 상세 DB 관리 — AI 답변에 필요한 데이터만 수집
4. ❌ 사용자 리뷰 작성 기능 — V3로 이관 (MVP 복잡도 관리)

### Success Metrics

| 지표 | Current (V1 前) | Target (V2 3개월) | 측정 방법 |
|------|---------------|-----------------|----------|
| 창원 맛집 DB 커버리지 | 50개 | **3,000개+** (24개 동) | Supabase count |
| MAU (B2C) | 0 | **5,000명** | 앱 로그 |
| AI 대화 완료율 (추천 클릭까지) | - | **70%+** | 세션 퍼널 |
| 추천 만족도 (👍/👎) | - | **80%+ 👍** | 인앱 피드백 |
| 앱스토어 리뷰 평점 | - | **4.3/5.0** | 스토어 |
| 매장 CTR (네이버로 이동) | - | **15%+** | 클릭 이벤트 |

## 4. Target Users & Segments

### Primary (핵심 타겟)
**"결정 장애 현지인/이주민 20-40대"**
- 창원/마산/진해 거주 or 자주 방문
- 약속 잡을 때마다 "어디 가지?" 고민

## 6. Solution Overview

### 전체 아키텍처

```
[크롤러 (기존 확장)]
  ├ 창원시 새로오픈 맛집 (일간)
  ├ 24개 동별 맛집 (주간)
  ├ 카테고리별 보완 (한식/일식/카페 등)
  └ 블로그/이미지/SNS 보강
        ↓
[Supabase restaurants 테이블 + 신규 보강 테이블]
        ↓
[AI 전처리 파이프라인 (Claude Haiku)]
  ├ 리뷰 → 감정/상황 태그 생성
  ├ 블로그 → AI 요약
  ├ 메뉴 이미지 → 가격대 추정
  └ 분위기 벡터 임베딩
        ↓
[Supabase pgvector (유사도 검색)]
        ↓
[AI 챗봇 API (Claude Opus/Sonnet + RAG)]
  ├ 사용자 쿼리 → 상황 파싱
  ├ 벡터 검색 + 구조화 필터
  └ 3-5개 추천 + 설명 생성
        ↓
[iOS/Android 앱 (Expo) — V1 기반 확장]
```

### 핵심 차별화: 수집할 데이터 (현재 + 신규)

| 카테고리 | 현재 수집 | V2에서 추가 수집 |
|---------|----------|----------------|
| **기본 정보** | 이름, 주소, 전화, 카테고리 | 영업시간(요일별), 휴무일, 브레이크타임 |
| **위치** | 위도·경도 | 구·동 파싱 완료, 주변 지하철/버스 정류장 |
| **가격** | - (신규) | **1인당 평균 가격대**, 최저가/최고가 메뉴 |
| **수용** | - (신규) | **좌석 수, 단체석 규모, 룸 유무** |
| **편의시설** | - (신규) | 주차/발렛, 키즈존, 유아의자, 반려동물, 휠체어, 와이파이, 콘센트 |
| **분위기** | 리뷰 키워드 (부분) | **분위기 벡터** (데이트/가족/회식/혼밥 적합도 AI 스코어) |
| **메뉴** | 상위 5개 | 전체 메뉴 + 가격, 매운맛/채식 태그 |
| **리뷰** | 키워드 리뷰, 방문자 리뷰수 | **리뷰 감정 분석** (긍·부정 비율), **최근 30일 트렌드** |
| **블로그** | 제목/스니펫 5개 | **AI 요약** (핵심 3문장), 방문 목적 분류 |
| **이미지** | 대표 이미지 1 | 메뉴판/인테리어/음식 분류, 밝기/분위기 태그 |
| **SNS** | - (신규) | 인스타 해시태그 노출 빈도 |
| **실시간** | - (신규) | 대기 시간 추정 (네이버 테이블링 연동) |
| **운영 상태** | - (신규) | **폐업 감지** (주간 체크) |

### AI 추천 엔진 설계

**Stage 1: 쿼리 이해 (Claude Haiku, <500ms)**
```
입력: "오늘 여자친구랑 분위기 좋은 곳 상남동"
출력: {
  intent: "recommendation",
  location: "상남동",
  company: "연인",
  mood: ["분위기좋은", "데이트"],
  meal_type: null,
  price_sensitivity: "medium"
}
```

**Stage 2: 하이브리드 검색**
- **구조화 필터**: 위치, 가격대, 편의시설 → Supabase SQL
- **벡터 유사도**: 분위기 벡터 × pgvector → 의미 검색
- 두 결과 교집합 → 상위 10개

**Stage 3: 랭킹 + 설명 생성 (Claude Sonnet, <3s)**
- 리뷰/블로그 데이터 컨텍스트로 주입
- 3-5개 선별 + "왜 이 곳인지" 자연어 설명
- **V1 언니픽 매장은 우선 노출** (단, 품질 기준 충족 시만)
