# PRD — 맛집 월드컵 16강전
**언니픽 앱 내 이상형 월드컵 형태 맛집 선택 게임**

> 버전 1.0 · 2026-04-18 · 작성자: bangju

---

## 1. Summary

맛집 월드컵은 언니픽 앱에서 등록된 가게 16곳을 이상형 월드컵 방식으로 겨루게 하는 게임이다.  
사용자는 상황 질문(예: "비 오는 날 혼자 밥 먹고 싶을 때")에 맞는 가게를 반복 선택해 최종 우승 맛집 1곳을 뽑는다.  
결과 화면에서 해당 가게 상세 페이지 또는 어울리는 BGM 플레이리스트로 자연스럽게 연결해 앱 체류 시간과 가게 노출을 동시에 높인다.

## 3. Background

### 3-1. 왜 이 기능인가?

언니픽의 `restaurants` 테이블에는 창원 맛집 데이터(이름, 카테고리, 평점, 리뷰, 메뉴, 이미지, 태그)가 쌓여 있다.  
그러나 앱에서 이 데이터를 **사용자가 직접 경험하는 인터랙티브 콘텐츠**로 소비하는 방법이 없다.  
단순 목록·지도 조회보다 게임 형식이 체류 시간과 재방문율을 크게 높인다는 것은 이미 국내 이상형 월드컵 트렌드(월드컵닷컴 등)로 검증됐다.

### 3-2. 왜 지금인가?

- `restaurants` 테이블에 충분한 데이터(100개 이상)가 축적됐다
- `custom_tags`, `category`, `review_keywords` 등 필터링 가능한 필드가 갖춰졌다
- Expo/React Native로 스와이프 기반 UI 구현이 쉬운 시점이다
- 경쟁 서비스(네이버 지도, 카카오맵)는 이런 게임형 기능이 없다

### 3-3. 핵심 가정

> 사용자는 "뭐 먹을까?" 결정 장애가 있다. 선택지를 2개로 좁혀주면 쉽게 선택한다.  
> 게임을 마친 후 "이 가게 가보고 싶다"는 행동으로 이어지게 하는 것이 핵심이다.

## 4. Objective

### 4-1. 목표

**사용자가 게임을 즐기는 동안 자연스럽게 창원 맛집을 발견하고, 실제 방문으로 이어지게 한다.**

- 사용자 가치: 결정 장애 해소 + 재미있는 맛집 탐색 경험
- 언니픽 가치: 앱 체류 시간↑, DAU↑, 맛집 데이터 노출 극대화
- 가게 가치: 게임을 통한 자연스러운 브랜드 노출

### 4-2. 성공 지표 (OKR)

| # | Objective | Key Result | 측정 시점 |
|---|-----------|------------|----------|
| O1 | 게임이 재미있어서 끝까지 한다 | 게임 완료율(16→결승) ≥ 60% | 출시 1개월 |
| O2 | 결과에서 가게로 이어진다 | 결과 화면 → 가게 상세 이동율 ≥ 40% | 출시 1개월 |
| O3 | 게임을 반복한다 | 1인당 월평균 플레이 횟수 ≥ 3회 | 출시 2개월 |
| O4 | 앱 체류 시간이 늘어난다 | 게임 플레이 세션 평균 시간 ≥ 3분 | 출시 1개월 |
| O5 | 공유가 일어난다 | 결과 공유 버튼 클릭율 ≥ 15% | 출시 2개월 |

| 상황 | 점심·저녁 메뉴 고민 중, 데이트 장소 탐색 중, 심심할 때 |

- "오늘 점심 뭐 먹지?" → 맛집 월드컵 열기 → "혼밥·가성비" 상황 선택 → 16강 진행 → 우승 가게 방문
- "친구랑 같이 해볼까?" → 각자 결과 비교 → 의견 일치 가게 방문

|------|-----------------|------------|
| 메뉴 결정 | 친구한테 물어봄, 배달앱 스크롤 | 15번 선택으로 자동 결정 |
| 새 가게 발견 | 네이버 지도 검색 | 게임 하다 보니 몰랐던 가게 발견 |

### 경쟁 대비 강점

- 네이버 지도·카카오맵: 게임 요소 없음, 단순 목록
- 이상형 월드컵 사이트(월드컵닷컴): 맛집 특화 아님, 앱 연동 없음
- 언니픽: **실제 로컬 데이터 + 게임 + BGM 플레이리스트 연결** 3가지 동시 제공

  │  [가게 상세 보기 →]                 │
  │  [어울리는 플레이리스트 듣기 🎵]    │

### 7-2. 핵심 기능 상세

#### A. 상황 질문 시스템

| 항목 | 내용 |
|------|------|
| 기본 상황 목록 | 10~15개 사전 정의 (혼밥, 데이트, 가족외식, 야식, 회식, 해장, 디저트 등) |
| 상황별 태그 매핑 | 각 상황 → `custom_tags`/`category` 필터 조합으로 DB 쿼리 |
| AI 랜덤 상황 | Gemini로 오늘 날씨·요일·시간대 기반 상황 문장 자동 생성 |
| 표시 방식 | 이모지 + 짧은 문장 (예: "🌧️ 비 오는 날 따뜻하게") |

**상황 → 태그 매핑 예시:**

| 상황 | 쿼리 조건 |
|------|----------|
| 혼밥 · 가성비 | `custom_tags @> '{혼밥}'` OR `category IN ('분식', '국밥', '백반')` |
| 데이트 · 분위기 | `custom_tags @> '{데이트}'` OR `rating >= 4.3` |
| 야식 · 배달 | `category IN ('치킨', '피자', '야식')` |
| 해장국 | `category ILIKE '%국%'` OR `custom_tags @> '{해장}'` |
| 디저트 | `category IN ('카페', '베이커리', '디저트')` |

#### B. 16강 대진표 구성

- 상황 태그로 필터된 가게 중 **평점순 상위 30개** 풀 생성
- 풀에서 **랜덤 16개** 선발 (같은 카테고리 편중 방지: 카테고리당 최대 4개)
- 대진표는 랜덤 시드로 매 게임마다 다르게 구성
- 이미지(`image_url`) 없는 가게는 카테고리별 기본 이미지로 대체

#### C. 대결 화면 UX

| 요소 | 내용 |
|------|------|
| 가게 카드 | 이미지(전체 높이), 이름, 카테고리 배지, 평점, 대표 리뷰 키워드 1개 |
| 선택 방법 | 카드 탭 또는 좌우 스와이프 |
| 애니메이션 | 선택된 카드 확대 + 탈락 카드 페이드아웃 (0.3초) |
| 진행 표시 | 상단 브래킷 아이콘 + "8강 3번째 대결" 텍스트 |
| 선택 취소 | 없음 (결정 장애 유도가 목적) |
| 소요 시간 | 15라운드 × 평균 5초 = 약 75초 (2분 이내) |

#### D. 결과 화면

| 요소 | 내용 |
|------|------|
| 우승 카드 | 가게 이미지 풀화면 + 트로피 이모지 + 이름/주소/평점 |
| CTA 1 | "가게 상세 보기" → `naver_place_url` 또는 앱 내 가게 상세 |
| CTA 2 | "어울리는 플레이리스트 듣기" → 상황 태그로 매핑된 BGM 플레이리스트 |
| CTA 3 | "다시 하기" → 같은 상황으로 재도전 |
| 공유 | 결과 이미지 카드(가게명 + "내 맛집 월드컵 우승!") → 카카오톡/인스타 공유 |

#### E. 플레이리스트 연결 매핑

| 상황 | 플레이리스트 무드 태그 |
|------|----------------------|
| 데이트 · 분위기 | `romantic`, `chill`, `jazz` |
| 혼밥 · 가성비 | `lo-fi`, `acoustic`, `relax` |
| 야식 · 신나게 | `upbeat`, `pop`, `hiphop` |
| 해장 · 아침 | `morning`, `calm`, `indie` |
| 디저트 · 카페 | `cafe`, `bossa nova`, `soft` |

### 7-3. 기술 설계

#### 데이터 흐름

```
[사용자 상황 선택]
    │
    ▼
[Supabase 쿼리]
  SELECT * FROM restaurants
  WHERE custom_tags @> '{태그}'
     OR category = '카테고리'
  ORDER BY rating DESC
  LIMIT 30
    │
    ▼ (앱 클라이언트에서)
[랜덤 16개 샘플링 + 카테고리 밸런싱]
    │
    ▼
[게임 상태 관리 (useState/useReducer)]
  { bracket: [], currentRound: 0, currentMatch: 0, winner: null }
    │
    ▼
[결과 → Supabase worldcup_results 테이블 저장]
```

#### 신규 DB 테이블

```sql
CREATE TABLE worldcup_results (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id     UUID REFERENCES users(id),
  situation   TEXT NOT NULL,              -- 선택된 상황 슬러그
  winner_id   TEXT NOT NULL,             -- naver_place_id
  bracket     JSONB DEFAULT '[]',        -- 전체 대진표 기록
  played_at   TIMESTAMPTZ DEFAULT now()
);
```

#### 어드민 연동 (Next.js)

- `/dashboard/restaurants` 에 "월드컵 노출 여부" 토글 추가 (`worldcup_enabled BOOLEAN`)
- 상황별 태그 매핑 설정 UI 추가
- 월드컵 플레이 통계 대시보드 (상황별 플레이 수, 가게별 우승 횟수)

### 7-4. 가정 (검증 필요)

| # | 가정 | 검증 방법 |
|---|------|----------|
| A1 | 창원 식당 이미지 품질이 게임에 충분하다 | 이미지 없는 가게 비율 확인 (목표: 80% 이상 이미지 보유) |
| A2 | 16개 풀이 상황별로 충분히 다양하게 구성된다 | 태그·카테고리 분포 분석 후 매핑 튜닝 |
| A3 | 2분 이내로 게임이 끝나야 이탈이 없다 | 초기 버전 완료율 모니터링 |
| A4 | 결과에서 플레이리스트로 이어지는 것이 자연스럽다 | A/B 테스트: CTA 순서 변경 비교 |
| A5 | 사용자가 결과를 공유하고 싶어한다 | 공유 버튼 클릭율 측정 |
