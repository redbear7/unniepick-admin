## 0. HotPepper 벤치마크 요약

### 일본 1위 맛집 앱이 된 이유 (2021년 기준 예약 사이트 1위, 사용자 만족도 57.5% 1위)

| HotPepper 차별화 요소 | 핵심 인사이트 |
|---------------------|-------------|
| **🎯 シーン検索 (상황 검색)** | 女子会·デート·歓送迎会·接待·合コン 등 *목적/상황* 중심 검색 — **언니픽 태그 시스템의 Root 아이디어** |
| **🔥 こだわり条件 (구체 조건)** | "치즈가 먹고싶다", "매운 거 먹고싶다" 같은 **음식 욕구 태그** |
| **🍽️ 세분화된 장르** | 한식>와쇼쿠>카이세키>료테이 같은 **3-4단계 계층 분류** |
| **⚡ 席押さえ (실시간 자리 선점)** | 현재 입장 가능 식당 + 즉시 자리 확보 |
| **💰 Ponta/d포인트** | 적립/사용으로 재방문 유도 — 쿠폰과 시너지 |
| **🎁 쿠폰 통합** | 앱 내 즉시 표시, 가게에서 바로 제시 |
| **🖼️ こだわり情報 (점주 자랑 포인트)** | 店長のおすすめ料理 / こだわりの空間 / 心をこめたおもてなし 등 **감성적 차별화 섹션** |
| **📊 검색 키워드 분석 (B2B)** | 점주가 "어떤 키워드로 내 가게를 발견했나" 확인 가능 |

### 언니픽 V2에 적용할 HotPepper DNA
1. **"シーン = 목적 태그"** 를 최우선 카테고리로 격상
2. **"こだわり条件 = 구체 욕구 태그"** 카테고리 신설
3. **"席押さえ = 실시간 대기 정보"** 기능 (네이버 테이블링 연동)
4. **"점장 자랑 섹션"** — 사장님이 직접 자랑 포인트 입력 → 언니픽 매장과 강력 시너지
5. **"검색 키워드 분석"** — 사장님 분석 도구 → B2B SaaS 유료 모델화 가능

## 1. Executive Summary

일본 HotPepper가 상황(シーン) 기반 검색으로 일본 시장을 석권한 DNA를, 창원 맛집에 적용한 **AI 태그 빅데이터 시스템**. 음악 트랙을 수십 개 태그로 분류해 상황별 플레이리스트를 만드는 언니픽 음악 시스템의 맛집 버전. 업체 1개당 평균 **30-50개 다차원 태그** × 12개 카테고리로 분류해, *"상남동 상견례 룸 한식"* 같은 복합 자연어 질문을 1초 안에 정확히 해석하고 추천한다.

**한 줄 가치**: *"네이버는 키워드, 카카오는 거리, 언니픽은 상황을 이해한다."*

## 2. 12개 태그 카테고리 (HotPepper + 한국 특화)

### 카테고리 구조 (우선순위 순)

```
⭐ 1. 목적/상황 (Scene) 🎯        ← HotPepper "シーン" 차용, 가장 중요
⭐ 2. 음식 욕구 (Craving) 🤤      ← HotPepper "こだわり条件" 차용
📍 3. 위치 (Location)
🏪 4. 편의시설 (Amenity)
🎨 5. 분위기 (Mood)
🍽️ 6. 메뉴 카테고리 (Genre)
💰 7. 가격대 (Price)
⏰ 8. 운영 시간 (Time)
👥 9. 고객층 (Audience)
🌶️ 10. 음식 속성 (Food Attr)
💬 11. 리뷰 감성 (Sentiment)
✨ 12. 점장 자랑 (Owner Picks) ← HotPepper "こだわり情報" 차용
```

### ⭐ 1. 목적/상황 (Scene) — 최상위 카테고리

#### 격식 있는 모임
- 상견례 · 프로포즈 · 기념일 · 결혼기념일 · 부모님생신 · 어버이날 · 어른접대

#### 일상 외식
- 데이트 · 가족외식 · 친구만남 · 혼밥 · 브런치 · 간식

#### 업무·회사
- 회식 · 접대 · 점심회의 · 송별회 · 환영회 · 승진축하 · 팀빌딩

#### 커뮤니티
- 동창회 · 돌잔치 · 아이생일 · 학부모모임 · 학교모임 · 학원수강끝나고

#### 특수 상황
- 소개팅 · 소개팅후술자리 · 이별후혼술 · 기분전환 · 비오는날 · 추운날 · 더운날

#### 시즌
- 벚꽃뷰 · 단풍뷰 · 크리스마스 · 연말 · 새해 · 복날

**★ 중요**: 각 태그에 **aliases (동의어)** 저장
예: `상견례` = `[양가상견례, 가족상견례, 부모님만남, 어른들인사]`

### ✨ 12. 점장 자랑 (Owner Picks) — HotPepper 핵심 차용

**사장님이 직접 입력하는 감성 차별화 섹션**:

```
사장님 자랑 입력 UI (언니픽 사장님 앱):

1. 🍳 店長のおすすめ — 대표 메뉴 1-3개 + 스토리
   "10년간 매일 아침 고르고 고른 국내산 한우만 씁니다"

2. 🏠 こだわりの空間 — 공간 자랑 1-3개
   "25년 된 한옥을 그대로 살린 서까래가 있습니다"

3. 💝 心をこめたおもてなし — 서비스 자랑 1-3개
   "상견례 손님께는 가족 사진 촬영 서비스를 무료로 드립니다"

4. 🎁 이번 달 프로모션 — 쿠폰/할인 자동 생성
```

**AI 추천 시 이 섹션이 "왜 이 가게인지" 설명에 직접 인용됨**

## 4. 데이터 모델 확장

### 점장 자랑 (Owner Picks) 테이블 추가

```sql
CREATE TABLE restaurant_owner_picks (
  id UUID PRIMARY KEY,
  restaurant_id UUID REFERENCES restaurants(id),
  pick_type TEXT,  -- 'signature_menu' | 'space' | 'service' | 'promotion'
  title TEXT,
  description TEXT,
  image_url TEXT,
  order_index INTEGER,
  verified BOOLEAN DEFAULT false,  -- 언니픽 운영자 검수
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### 실시간 상태 테이블 (席押さえ용)

```sql
CREATE TABLE restaurant_live_status (
  restaurant_id UUID PRIMARY KEY REFERENCES restaurants(id),
  is_open_now BOOLEAN,
  wait_estimate_minutes INTEGER,
  available_tables INTEGER,
  last_updated TIMESTAMPTZ DEFAULT now(),
  source TEXT  -- 'naver_tabling' | 'manual' | 'estimate'
);
```

### 쿠폰 테이블 (HotPepper 쿠폰 기능 차용)

```sql
CREATE TABLE restaurant_coupons (
  id UUID PRIMARY KEY,
  restaurant_id UUID REFERENCES restaurants(id),
  title TEXT,
  description TEXT,
  discount_type TEXT,  -- 'percent' | 'amount' | 'free_item'
  discount_value NUMERIC,
  conditions TEXT,
  valid_from DATE,
  valid_until DATE,
  max_uses INTEGER,
  used_count INTEGER DEFAULT 0,
  is_unniepick_exclusive BOOLEAN DEFAULT true,  -- V1 매장 독점 쿠폰
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## 6. 사장님 대시보드 (HotPepper B2B 차용)

V1 언니픽 매장 사장님만 접근 가능:

### 기능
1. **내 가게 태그 관리** — AI가 붙인 태그 + 수동 보정
2. **점장 자랑 입력** — 대표메뉴/공간/서비스 스토리
3. **실시간 상태 토글** — "지금 자리 있어요/대기중이에요"
4. **쿠폰 발행** — 기간/조건 설정
5. **⭐ 검색 키워드 분석** — "어떤 키워드로 내 가게가 노출됐나"
   - "상견례"로 100회 노출, 10회 클릭
   - "룸 있는 한식" 으로 50회 노출, 8회 클릭
   - → 약점 파악: 가격대 키워드 유입 적음 → 가성비 쿠폰 발행 제안
6. **경쟁 분석** — 같은 동/카테고리 TOP 3 가게 태그 비교

**★ 이 기능은 B2B SaaS 유료 플랜으로 확장 가능** (HotPepper 유료 계약과 동일 모델)

## 7. AI 태깅 파이프라인 (강화)

### 소스별 태깅

| 소스 | 생성 태그 카테고리 |
|------|-----------------|
| 리뷰 키워드 | 리뷰 감성 |
| 블로그 리뷰 (Claude Haiku) | **목적/상황** ⭐ + 분위기 + 고객층 |
| 메뉴 데이터 + 이미지 | 메뉴 장르 + 가격대 + 음식 속성 |
| 편의시설 크롤링 | 편의시설 |
| 영업시간 크롤링 | 운영 시간 |
| 주소 파싱 | 위치 |
| 사장님 입력 | 점장 자랑 + 일부 편의시설 |
| 블로그 맥락 | 음식 욕구 ("비 오는 날 생각나는 국물") |

### 프롬프트 예시 (목적/상황 태깅)

```
블로그 리뷰를 읽고 이 식당이 어떤 상황에 어울리는지 태그를 선택하세요.

식당명: ○○한정식
블로그 제목: "양가 상견례 후기"
본문: "부모님 모시고 다녀왔습니다. 조용한 VIP룸에서..."

카테고리: [상견례, 데이트, 회식, 가족외식, 혼밥, 친구모임, 프로포즈, 기념일, ...]

JSON 응답:
{
  "primary_scene": "상견례",
  "secondary_scenes": ["가족외식", "부모님생신"],
  "confidence": 0.95,
  "evidence": "양가 상견례 후기, 부모님 모시고, 조용한 VIP룸"
}
```

## 8. 구현 로드맵

### Phase A — 태그 마스터 + UI (3주)
1. 12개 카테고리 × 250+ 태그 시드 데이터
2. 태그 관리 대시보드 (추가/병합/동의어)
3. 맛집별 수동 태깅 UI

### Phase B — AI 태깅 엔진 (3주)
1. 소스별 프롬프트 설계 (10개 소스)
2. 배치 파이프라인 (Claude Haiku)
3. 증분 태깅 (신규 크롤링 자동)
4. 검수 큐

### Phase C — 사장님 대시보드 (2주) ⭐ HotPepper 차용
1. 점장 자랑 입력 UI
2. 실시간 상태 토글
3. 검색 키워드 분석
4. 쿠폰 발행

### Phase D — 쿼리 API (2주)
1. 자연어 → 태그 파서
2. 다태그 매칭 SQL
3. 랭킹 + 설명 생성
4. 쿠폰/자랑 섹션 결합

### Phase E — 사용자 앱 UI (3주)
1. HotPepper 스타일 홈 (시나리오 칩)
2. 채팅 검색
3. 카드형 결과 (태그 칩 + 사장님 자랑 + 쿠폰)
4. 실시간 대기 정보

### Phase F — 런칭 + 개선 (2주)
1. 베타 30명
2. 피드백 기반 태그 보정
3. 창원시 PR

**Total: 15주 (3.5개월)**

## 12. 창원 특화 포인트

HotPepper가 도쿄/오사카 같은 대도시 기준인 반면, 언니픽은 창원 특수성 반영:

1. **NC파크 연동**: 야구 경기 전후 특화 검색 ("NC경기 후 뒷풀이")
2. **진해 군항제/벚꽃**: 시즌 태그 ("벚꽃뷰", "군항제근처")
3. **창원공단**: 직장인 점심/회식 특화 ("공단근처", "점심특선")
4. **마산 어시장**: 해산물 본고장 특화 ("마산회", "아귀찜")
5. **경남대/창원대**: 대학생 상권 ("대학생가성비", "과회식")
