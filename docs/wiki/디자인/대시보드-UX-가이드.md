## 다크모드 컬러 토큰 (Tailwind 변수)

```css
/* globals.css */
--color-primary: #ffffff;       /* 주 텍스트 */
--color-secondary: #e8e8e8;      /* 부 텍스트 */
--color-muted: #888888;          /* 힌트/라벨 */
--color-card: #1a1a1a;           /* 카드 배경 */
--color-sidebar: #141414;        /* 사이드바 */
--color-fill-subtle: #252525;    /* 미묘한 배경 */
--color-border-main: #2a2a2a;    /* 메인 테두리 */
--color-border-subtle: #1f1f1f;  /* 은은한 테두리 */

--accent: #FF6F0F;               /* 언니픽 오렌지 */
```

## 컴포넌트 표준

### 버튼
```
[ Primary ]       → bg-[#FF6F0F] text-white
[ Secondary ]     → bg-card border border-border-main
[ Ghost ]         → text-muted hover:text-primary
```

### 입력 필드
```
┌──────────────────────────────┐
│ 🔍 placeholder                │  bg-fill-subtle
└──────────────────────────────┘   border focus:border-[#FF6F0F]
```

### 칩 (Tag/Filter)
```
[ 태그 ]          ← 기본 (회색 테두리)
[ 태그 ✓ ]       ← 활성 (오렌지)
[ 태그 23 ]      ← 카운트 포함
```

### 카드
```
┌─────────────────────┐
│ [이미지]             │  rounded-xl
│                      │  border border-border-main
│ 제목 🏷️ 칩           │  p-4 space-y-3
│ 부가 정보             │  hover:border-[#FF6F0F]/50
└─────────────────────┘
```

### 상태 배지
- `NEW`: `bg-green-500/15 text-green-400`
- `실행중`: `bg-amber-500/15 text-amber-400` + 스피너
- `실패`: `bg-red-500/15 text-red-400`
- `완료`: `bg-blue-500/15 text-blue-400`

## 상호작용

### 폴링 패턴
- running 상태가 있을 때만 5초 간격
- 상태 변경 감지 시 자동 중지
- 컴포넌트 언마운트 시 clearInterval

### 확인 다이얼로그
- 삭제는 `confirm()` 필수
- 되돌릴 수 없는 작업은 두 단계 확인

### 로딩 상태
- 초기: 중앙에 스피너 + "로딩 중..."
- 빈 상태: "데이터 없음" 메시지 + 다음 액션 힌트
- 실패: 오류 메시지 + 재시도 버튼

## 반응형 정책

- 모바일 최적화는 **우선순위 낮음** (어드민은 데스크톱 주)
- 최소 폭 1024px 기준 설계
- 태블릿은 2-column → 1-column 변환

## 접근성

- 모든 버튼에 `title` 또는 aria-label
- 색상만으로 상태 표시 금지 (아이콘 병행)
- 키보드 포커스 링 유지
- Enter/Escape 기본 동작 지원

## 빈번한 실수

❌ 라이트모드 값을 다크에 재사용 (흰 배경에 보이는 회색이 다크에선 안 보임)
✅ 항상 `text-primary/secondary/muted` 토큰 사용

❌ 고정 px 마진 (반응성 약함)
✅ Tailwind `gap-*`, `space-y-*` 활용

❌ confirm() 없는 DELETE
✅ 모든 삭제에 확인 필수
