# 빅데이터 관리 오픈소스 비교

> 언니픽 창원맛집 빅데이터 관리를 위한 오픈소스 솔루션 비교 분석.
> **최종 선택**: Metabase (BI) + Directus (데이터 관리)

## 요구사항

1. Supabase (PostgreSQL) 직접 연결
2. 맛집 데이터, 리뷰 키워드, 태그, 메뉴, 블로그 리뷰 처리
3. 시각화 (차트, 그래프, 지도)
4. Next.js 임베드 가능
5. Self-hosted 우선

## 비교 요약

| 도구 | PostgreSQL | 지도 시각화 | 태그/관계 관리 | Next.js 임베드 | 배포 | 적합 용도 |
|------|----------|-----------|--------------|--------------|------|----------|
| **Metabase** ⭐ | ✅ 네이티브 | ✅✅✅ (Pin/Region) | ⚠️ 쿼리 기반 | ✅✅ SDK 지원 | Docker 간단 | **BI/대시보드** |
| **Directus** ⭐ | ✅ 스키마 자동 인식 | ❌ | ✅✅✅ (M2M) | ⚠️ API 연동 | Docker 간단 | **데이터 관리** |
| Apache Superset | ✅ | ✅ | ❌ | iframe 복잡 | 설정 복잡 | 고급 BI |
| Grafana | ✅ | ⚠️ 제한 | ❌ | iframe | Docker | 모니터링 |
| Redash | ✅ | ⚠️ | ❌ | iframe | Docker | 쿼리 공유 |
| NocoDB | ✅ | ❌ | ✅✅ | API만 | Docker 최간편 | Airtable 대안 |
| Evidence | ✅ | ⚠️ | ❌ | 정적 사이트 | 복잡 | BI as Code |

## 1위: Metabase — BI/시각화

### 선택 이유
- **지도 시각화 최강** (Pin Map, Region Map, GeoJSON 지원) → 맛집 위치 표시에 필수
- PostgreSQL 네이티브 드라이버 Docker 이미지 내장
- **한국어 완벽 지원** (UI + 로케일)
- **Next.js SDK + iframe JWT 임베드** 둘 다 지원
- SQL 없이도 차트/대시보드 생성
- 무료, 완전 오픈소스

### 한계
- 관계 관리는 쿼리 기반 (구조화 편집 어려움)
- 데이터 편집보다 조회/분석 특화

### 사용처 (언니픽)
- 카테고리별 업체 분포
- 리뷰 수 TOP 10
- 지도 분포
- 키워드 리뷰 트렌드

## 2위: Directus — 데이터 관리

### 선택 이유
- **PostgreSQL 스키마 자동 인식** → Supabase 테이블 즉시 노출
- **M2M 관계 완벽 지원** (맛집 ↔ 태그, 리뷰 ↔ 키워드)
- REST + GraphQL API 자동 생성
- 관리자 UI, 권한 관리, 파일 처리 내장
- Docker 배포 10~20분

### 한계
- 시각화 레이어 부재 (Metabase 병용 필요)
- 기업 매출 $5M 이상 시 상용 라이선스 필요 (우리는 무관)

### 사용처
- 맛집 데이터 직접 편집/보정
- 태그 마스터 관리
- 사진/파일 업로드
- 크롤링 이후 수동 검수

## 3위: NocoDB — Airtable 대안

### 장점
- 가장 간편한 설정 (10분 이하)
- Grid/Kanban/Gallery/Form 뷰
- Linked Records

### 한계
- **지도 미지원** → 맛집 서비스에 치명적
- 차팅 제한적
- SUL 라이선스 전환 (내부 사용만 무료)

## Apache Superset — 탈락 사유
- 강력한 BI지만 **배포/설정 복잡** (1인 개발 부담)
- Metabase가 동일 기능을 더 쉽게 제공

## 최종 아키텍처

```
┌─────────────────────────┐
│   Supabase (PostgreSQL) │  ← 데이터 원본
│   restaurants 등         │
└─────────────────────────┘
     ▲            ▲
     │            │
┌──────────┐  ┌──────────┐
│ Metabase │  │ Directus │
│ (BI)     │  │ (편집)    │
└──────────┘  └──────────┘
     ▲            ▲
     │            │
┌──────────────────────────┐
│ 언니픽 Next.js 대시보드    │
│ /dashboard/restaurants    │  ← Supabase 직접 쿼리
│ /dashboard/restaurants/   │
│     analytics (Metabase)  │  ← iframe 임베드
└──────────────────────────┘
```

## 역할 구분 (비유)

| 도구 | 비유 | 역할 |
|------|------|------|
| **Metabase** | 엑셀 차트/피벗테이블 | 데이터를 **보고 분석** |
| **Directus** | 노션/에어테이블 | 데이터를 **편집하고 관리** |
