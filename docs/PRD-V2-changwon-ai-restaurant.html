PRD: 언니픽 V2 · 창원 AI 맛집 추천 시스템

Author: bangju Date: 2026-04-17 Status: Draft Target: 언니픽 앱 V2 (V1 런칭 후 추가) Tagline: "창원 최초 AI 맛집 추천 시스템"

1. Executive Summary

창원 지역 맛집 50만+ 데이터 포인트를 축적해 상황/감정/동행 맥락을 이해하고 대화로 맛집을 추천하는 AI 챗봇 서비스. 네이버·카카오 지도가 제공하지 못하는 "오늘 데이트 분위기 좋은 이자카야" 수준의 맥락 기반 추천을 창원 지역 최초로 상용화한다.

한 줄 가치: 지도 앱이 답을 찾게 하는 동안, 언니픽은 대화로 "딱 맞는 곳"을 제안한다.

2. Background & Context

현재 시장의 한계

서비스 한계
네이버 플레이스 키워드 검색 기반, 맥락 이해 불가 ("데이트 분위기 좋은" → 안 나옴)
카카오맵 거리/평점 정렬 중심, 상황별 추천 부재
다이닝코드/블루리본 서울/부산 중심, 창원 커버리지 취약
미쉐린 창원 미포함

언니픽만의 강점

  • B2B 매장 네트워크 보유 (V1): 매장 사장님 연결 고리 → 쿠폰/예약/정보 업데이트 협력 가능
  • 1인 개발자의 민첩성: 대기업 플랫폼 대비 빠른 창원 특화 기능 반영
  • 이미 구축된 빅데이터 파이프라인:
    • 크롤러 (일 50+ 업체, 24개 동 대응 가능)
    • Supabase + Metabase + Directus
    • 리뷰/블로그 감성 데이터
    • 텔레그램 신규 업체 알림

V1 (현재) vs V2 (목표)

V1 (2026-04 런칭) V2 (Target: 2026 Q3)
서비스 BGM 스트리밍 + TTS (B2B) B2B + 창원맛집 탭 (B2C)
사용자 매장 사장님 사장님 + 일반 소비자
수익모델 구독료 구독 + 맛집 광고/쿠폰 수수료

3. Objectives & Success Metrics

Goals

  1. 창원 1위 맛집 추천 앱 포지셔닝 (앱스토어 "창원 맛집" 키워드 1페이지 노출)
  2. 대화 UX로 차별화: 버튼/필터가 아닌 자연어 대화로 90%+ 추천 성공 경험 달성
  3. B2B→B2C 확장: V1 사장님 매장을 AI 추천 대상에 연동 (언니픽 매장 우선 노출)

Non-Goals

  1. ❌ 전국 서비스 — 창원/마산/진해만 집중 (로컬 깊이 우선)
  2. ❌ 배달/예약 직접 처리 — 네이버/카카오 링크로 연결
  3. ❌ 일반 음식점 상세 DB 관리 — AI 답변에 필요한 데이터만 수집
  4. ❌ 사용자 리뷰 작성 기능 — V3로 이관 (MVP 복잡도 관리)

Success Metrics

4. Target Users & Segments

Primary (핵심 타겟)

"결정 장애 현지인/이주민 20-40대"

  • 창원/마산/진해 거주 or 자주 방문
  • 약속 잡을 때마다 "어디 가지?" 고민
  • 네이버/인스타 맛집 피드 피로감
  • 키워드 검색이 잘 안 되는 상황 (예: "비 오는 날 저녁 혼자 먹기 좋은 곳")

Secondary

  • 창원 여행객: 출장/관광으로 방문한 외지인
  • V1 매장 사장님: 본인 매장 AI 노출 확인

사용자 시나리오 (Jobs-to-be-done)

상황 현재 고통 V2가 주는 가치
여자친구랑 분위기 좋은 데이트 인스타 피드 1시간 스크롤 "상남동에 루프탑 이자카야 3곳 추천 (20-30대 데이트 비율 높음)"
회사 회식 40명 한식 전화 뺑뺑이 "단체석 40명+ 가능한 창원 한식 4곳, 방 개수 포함"
아이와 브런치 키즈존 정보 불명확 "키즈존/유아의자 보유 + 브런치 메뉴 있는 곳 5곳"
비 오는 날 감성 카페 주관적 판단 어려움 "창문석 + 잔잔한 BGM 리뷰 많은 카페 3곳"
혼밥 가성비 눈치 보임 "1인석/카운터석 있고 8천원 이하 점심 5곳"

5. User Stories & Requirements

P0 — Must Have (MVP)

# User Story Acceptance Criteria
1 사용자로서 자연어로 "오늘 데이트 분위기 좋은 곳"이라고 입력하면 3-5개 맛집을 받고 싶다 응답 10초 이내, 각 카드에 이름·카테고리·평점·대표사진·"왜 추천하는지" 설명 1줄
2 사용자로서 구/동 선택 시 해당 지역 맛집만 보고 싶다 칩 UI로 24개 동 필터, 결과 즉시 갱신
3 사용자로서 추천받은 맛집을 네이버지도/전화로 바로 연결하고 싶다 딥링크, iOS/Android 모두 동작
4 사용자로서 추천 결과에 👍/👎 피드백을 남기고 싶다 익명 가능, DB 저장 (후속 추천 개선용)
5 사용자로서 새로오픈 맛집을 빠르게 확인하고 싶다 홈 상단 "이번 주 새로오픈" 섹션, 일 1회 자동 갱신

P1 — Should Have

# User Story Acceptance Criteria
6 사용자로서 후속 질문 "더 가성비 있는 곳"으로 대화를 이어가고 싶다 이전 대화 맥락 유지, 필터 누적
7 사용자로서 지도에서 맛집 위치 한눈에 보고 싶다 네이버맵 SDK 임베드, 핀 클릭 시 상세
8 사용자로서 즐겨찾기/방문 기록을 저장하고 싶다 무로그인 로컬 저장, 로그인 시 동기화
9 사장님으로서 본인 매장 정보를 보정 요청할 수 있다 V1 사장님 계정 연동, 수정 요청 → 관리자 검토

P2 — Nice to Have / Future

# User Story Acceptance Criteria
10 사용자로서 "친구가 방문한 곳" 같은 소셜 추천 V3: 친구 추가 기능 필요
11 사용자로서 예약 가능 맛집 바로 예약 네이버예약 OAuth 연동
12 매장 쿠폰 자동 발급/사용 V1 쿠폰 시스템 재활용

AI 추천 엔진 설계

Stage 1: 쿼리 이해 (Claude Haiku, <500ms)

입력: "오늘 여자친구랑 분위기 좋은 곳 상남동"
출력: {
  intent: "recommendation",
  location: "상남동",
  company: "연인",
  mood: ["분위기좋은", "데이트"],
  meal_type: null,
  price_sensitivity: "medium"
}

Stage 2: 하이브리드 검색

  • 구조화 필터: 위치, 가격대, 편의시설 → Supabase SQL
  • 벡터 유사도: 분위기 벡터 × pgvector → 의미 검색
  • 두 결과 교집합 → 상위 10개

Stage 3: 랭킹 + 설명 생성 (Claude Sonnet, <3s)

  • 리뷰/블로그 데이터 컨텍스트로 주입
  • 3-5개 선별 + "왜 이 곳인지" 자연어 설명
  • V1 언니픽 매장은 우선 노출 (단, 품질 기준 충족 시만)

8. Timeline & Phasing

Phase 1 — 데이터 확보 (M1-M2, 2026-05~06)

  • 24개 동별 크롤링 스케줄러 전환 (주간)
  • 신규 데이터 포인트 수집: 영업시간, 가격대, 편의시설
  • 리뷰 감정 분석 파이프라인 (Claude Haiku)
  • 블로그 AI 요약 배치 작업
  • pgvector 확장 + 분위기 임베딩
  • 목표: 3,000개 맛집 × 30+ 데이터 포인트

Phase 2 — AI 엔진 MVP (M3, 2026-07)

  • 쿼리 파싱 API (Haiku)
  • 하이브리드 검색 (SQL + Vector)
  • 랭킹 + 설명 생성 (Sonnet)
  • 대시보드 프롬프트 실험실 (내부 테스트용)

Phase 3 — 앱 통합 (M4, 2026-08)

  • Expo iOS/Android 창원맛집 탭 추가
  • 채팅 UI + 카드형 결과
  • 피드백 수집 (👍/👎)
  • 즐겨찾기 (로컬 저장)
  • 베타 테스트 (30명 내부)

Phase 4 — 런칭 (M5, 2026-09)

  • "창원 최초 AI 맛집 추천" PR
  • 창원시 SNS/블로거 협력
  • V1 사장님 매장 연동 (우선 노출)
  • 앱스토어 출시

Phase 5 — 개선 (M6+)

  • 피드백 기반 추천 모델 튜닝

9. Risks & Mitigation

리스크 확률 영향 대응
네이버 크롤링 정책 변경/차단 중 높음 카카오/다이닝코드 백업 소스 확보
